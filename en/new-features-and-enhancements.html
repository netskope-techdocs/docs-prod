Advanced Analytics

New Dimension

The following new dimensions are available with this release:

  • forward_to_proxy_profile

  • netskope_pop (for SkopeIT)

Advanced Analytics

Country Name

With this release, country name is available in the Source country and Destination country fields.

Behavior Analytics

Anomalies rules are now read only

With this release, Anomalies rules are now read only and can no longer be edited or changed. They should only be disabled as they are migrated to policies in Behavior Analytics. Until they are disabled, alerts will be seen for both Anomalies rules and Behavior analytics.

Behavior Analytics

Enhancements to the user related columns

User name is no longer embedded in the Behavior Analytics incident description. The Acting User column is always shown next to the incident description, and is subject to obfuscation.

CASB Inline Protection

Instance ID support synthetic requests/responses

We have added Instance ID support to CLI activities for the following services:

  • AWS S3

  • EC2

  • SNS

  • Lambda

  • RDS

Data Protection

Multi-label image classification

In this release, three additional ML based Image classifiers have been added.

Admins can build out a file classification policy for:

  • Healthcare ID Cards

  • Personal and banker's checks

  • International Driver License

The International Driver license classifier is a super set of the currently existing US driver license classifier.

Data Protection

File Filter file extension

File Profile has been updated to allow file type search based on file extensions. This makes it easier to search for and specify file types of interest in a File Profile.

Directory Services

Netskope Adapters

The Netskope Adapter (NS Adapter) has been tested to ensure compatibility with the current cloud platform. Its version number has been updated to confirm this compatibility. No other changes have been made to the NS Adapter in this release.

Netskope for IaaS

Multi-select filters for Compliance Standard

Enhanced the Reports Compliance Standard Filter to multi-select as follows:

  • Changed Compliance > Rules/Resources/Raw Findings Compliance Standard Filter to multi select

  • Disabled Compliance > Rules/Resources/Raw Findings "Sort by" Compliance unless only one Compliance Standard is selected

NG SWG / CASB

Chromebook verified access

The Chromebook verified access feature provides the ability to differentiate enrolled Chromebook logins versus unenrolled Chromebook logins. Admins can block unenrolled logins. This is for the reverse-proxy method only.

Contact Support to enable this feature in your account.

Other Services (UI, API etc.)

Add new categories for CASB apps

New categories have been added for CASB apps for better mapping of the application with its closest category.

  • Chat, IM, and other communication

  • Personal Sites and Blogs

  • Professional Networking

  • Streaming and Downloadable Video

  • Shopping

  • Remote Access

Private Access

Network Events support for Advanced Analytics

We have added support for Private Access Network Events to be included in the Advanced Analytics reports.

Private Access

Network Events displays the Private App name in SkopeIT

Private Access Network Events will now specify the Private App name in the SkopeIT Application Name field.

Previously, the Application Name field contained the hostname of the Private App being accessed. This hostname value has been moved to the Destination Host field.

Private Access

Improved Private Access Client logging

Private Access Client logging has been updated to match the logging level of the Netskope Client and default to Info level logging.

Private Access

Docker host networking enhancement

The Private Access Publisher configuration uses docker host networking. To avoid contention with some private app configurations, this update disables docker bridge networking on the Publisher which could interfere with some Private App IP ranges.

Private Access

Log level configuration enhancement

This Private Access Publisher release includes additional log level configuration functionality to assist with troubleshooting.

Private Access

Troubleshooting functionality enhancement

The Private Access Publisher has been enhanced with troubleshooting functionality to assist customers with investigating deployed instances.

REST API

Public Cloud API for Storage Constraint Profile

Admins are now able to view bucket list, replace bucket list, append to bucket list, delete from bucket list, and modify bucket list using the ‘storage_constraint_profile’ API if there is more than one custom storage constraint profile.

In the past, admins were not able to perform any of the previously mentioned actions if they created more than one custom storage constraint profile via the UI.

This rule was added to ensure that the backend system did not get overwhelmed. Now that the backend system has matured, the rule has been removed.

Steering

Netskope Client log level

Netskope Client log level can be controlled using a new log option available under the Web UI client configuration.

Steering

Netskope Client Golden Release

Release 84 is the current golden release of the Netskope Client.

Release 84 binaries will be available soon, from the Support page.

For information about supported platforms and deployment instructions, see the Netskope Client Installation Guide.

Steering

Netskope Client UPN file preference

Netskope Client adds the ability to get the UPN from the preference file instead of the Active Directory.

Admins can specify the preference file name so the Netskope Client reads the UPN from the file. The Netskope Client gets the UPN from the Active Directory if the preference file is not specified.

Web UI

Last Updated Field

With this release, the "Last Updated" field in all Introspection pages is no longer visible.

However, users can still see the "Last Updated" field in the Files Page and dashboard.
