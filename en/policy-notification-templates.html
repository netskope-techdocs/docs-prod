Policy Notification Templates

You can use the templates to customize different notification pages shown to the users, plus you can upload your company logo so it appears in all the pages. All of these templates can be modified to suit your needs.

These are the types of templates you can use to help guide your users:

  • Customize block and alert notifications

  • Customize emails

  • Add custom images

You can create new templates, use the default templates, and preview templates by clicking Policies > Templates. Select the type of template you want to create. After creating these templates, you can choose to use them when sending policy notifications. You can further customize the template by adding justifications, text, configure action buttons, and redirect users to specific URLs.

You can customize the templates by adding variables, when available, for subject and message. Options include: Application name, Category name, URL accessed by user, Domain accessed by user, Activity performed, Triggered policy name, User Email/File Owner, Application Instance, and File name. 

User Notification Template

User notification templates enable you to block a user action and/or send an alert to the user. Templates can be customized to provide specific information and options in an alert or block notification. You can provide a justification dropdown list for your users and also display a description box to provide a justification. Optionally, you can make the justification text optional. There are also options for users to include, stop, or proceed after providing a justification. In addition, you can use the Insert button to add Netskope template variables and custom images.

On the User Notification page, click New Template.

Enter a name, title, and message, and then choose the options you want, like block or alert, justification, etc. Some options, like Message, Subtitle, and Redirect End Users to the Following URL Automatically, have variables you can select from a dropdown list. When finished, click Save.

To open the Settings dialog box, click the gear icon on the User Notification page.

All templates use these global settings.

Delivery & Timeout Settings:

Notification Delivery Method: Select how to notify the user, using the Client (when installed and enabled), or by browser.

  1. Browser based delivery: Notification will be delivered to the browser in the following two cases.

    1. When a browser based longpoll connection is established, notification pop-ups for all activities will be delivered to the browser. When longpoll connection is not available for browse activity, an attempt is made to deliver notifications in the body of browser.

    2. When notification delivery is not possible to the browser (as a backup), if the client is installed and available, notifications will be delivered to the client. If the client is also not available, no notification will be delivered and the default action will be block for both 'Block' and 'User Alerts'.

  2. Client based delivery: Notifications will be delivered to the client for all activities if the client is installed and available.

Notification Timeout: Enter the number of seconds (up to 600) that you want the pop-up notification to be shown to the user when you set 'User Alert' or 'Block' as a policy action. The default is 60 seconds.

Mute Settings:

The Mute settings control how long repeat notifications are suppressed after the initial notification.You can mute notifications for DLP & Threat Protection Policies and all other policies. The range is from one minute to 12 hours. This option is only applicable if the notification delivery method is 'Client'.

Parameters used for mute consideration:

  1. For DLP & Threat Protection Policies:

    1. Repeat notifications are muted when the following parameters are the same: User, App, Activity, and File Name

  2. For all other policies:

    1. Block notifications are muted when the following parameters are the same: User, App, Activity, and File Name

    2. For User Alert: User and App parameters are considered

Note

In case of 'Other Policies' User Alert, if the user clicks 'Cancel', the cancel suppression will be applicable only for one minute and not for the configured duration. However, suppression with configured values will be honored for the proceed action.

Email Notification Template

Email templates can be customized by directly editing the HTML in the Create Email Notification Template window. Optionally, you can use the Insert button to add Netskope template variables.

Note

The Email Sender, Email Recipient, and Email Type: Body or Attachment template variables apply to Gmail and Microsoft Outlook SaaS apps only.

Custom Images

Custom images, like your company logo, can be added using the Upload Custom Image window. These images can be added to all the other templates described in the following sections.

Acceptable logo sizes are:

  • 48px for small

  • 64px for medium

  • 128px for large

This logo size is applicable only if the company logo / custom image height is greater than the selected size. If the original logo size is less than the selected size, it will display the original image. For example, if the uploaded image height is 55px and if the selected size is medium (64px), the system will display the original image only (55px). However, if the selected size is small (48px), the system will shrink the image and its size will be 48px.

Single Sign-On

This template is available for Client Enforcement using single sign- on.

Client enforcement allows you to implement cloud security by making sure the user traffic is always steered through Netskope. You can implement client enforcement by integrating with single sign on or by using Netskope as the SAML proxy.

By default, when you are enforcing a client, users are automatically redirected to the download page to download and install the client.

You can also choose to show a custom page to the users to provide more information on the company security policy without redirecting them to the download page.
