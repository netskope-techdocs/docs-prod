Netskope Client Command Reference

This section lists various options used long with the MSIEXEC command. The MSIEXEC command is used to mass deploy Netskope Client (MSI packages) on Windows devices.

The format of MSIEXEC command:

Generic format: msiexec /i installmode=<option> <MSClient file> token=<organization ID> host=<host> /l*v %PUBLIC%\nscinstall.log

Single Mode Installation

>msiexec /I NSClient.msi [InstallMode=idP [tenant=<tenant-name> domain=<domain>]]

If your tenant URL is mytenant.example.com, then, tenant-name= mytenant and domain = example.com

If your tenant URL is mytenant.eu.example.com, then, tenant-name=mytenant and domain=eu.goskope.com

Example

msiexec /I NSClient.msi installmode=idP tenant=example domain=eu.goskope.com

Multi-user Mode Installation

>msiexec /I NSClient.msi installmode=idP tenant=<tenant-name> domain=<domain> mode=peruserconfig [userconfiglocation=<path>]

Example:

msiexec /I NSClient.msi installmode=idP tenant=example domain=eu.goskope.com mode=peruserconfig userconfiglocation=C:\Users\^%USERNAME^%\Netskope

Arguments used with MSIEXEC command

The following are the list of arguments commonly used along with the MSIEXEC command when deploying Netskope Client.

/i

Optional Command. Refers to normal installation type.

<NSClient-File>

The name of the Netskope Client installer file. The latest Netskope client installer is available from the Support portal.

Token

Enter your organization ID here. To find your organization ID.

  1. Login to your tenant with admin credentials.

  2. Go to Settings > Security Cloud Platform > MDM Distribution.

  3. Locate your Organzation ID under Create VPN Configuration section.

installmode=idP

This parameter specifies that client deployment mode is IdP Provisioning.

Host

The addon URL of your tenant. For example: if your URL is seiu.goskope.com, then host = addon-seiu.goskope.com
