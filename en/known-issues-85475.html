Known Issues

The following known issues are included in this release.

Issue Number

Issue Description

Workaround

113634

The restart interface command displays the underlying names of interfaces such as eth0, eth1 in the options instead of the interfaces such as dp1, dp2, and mp.

There is no workaround.

114231

When processing some log files, a benign error message is received in /opt/ns/log/logwatcher00.log.

Error submitting packet: [Errno 111] Connection refused, dropping the packet and closing the socket

The error message does not impact the functionality of the log upload and can be ignored.

114027

Unable to add destination network on an appliance with radius server configuration using the following commands.

add interface inbound destination-networks
set interface inbound destination-networks 0 networks <destination-ip-address>
save

Use radius server without the use of destination-networks. This means that radius-server can be configured as usual in the CLI. The “destination-networks” command should not be configured under interface configuration.

113998

On an appliance configured with radius server, no error message is displayed when the same network or IP address is configured twice.

Use radius server without the use of destination-networks. This means that radius-server can be configured as usual in the CLI. The “destination-networks” command should not be configured under interface configuration.

113996

On an appliance configured with radius server, overlapping network/ip configs are currently allowed.

Use radius server without the use of destination-networks. This means that radius-server can be configured as usual in the CLI. The “destination-networks” command should not be configured under interface configuration.

113626

Unable to set dp1 interface using destination network on an appliance with radius server configuration. The following error is displayed when running set interface dp1 destination-networks.

"err": "Save aborted. Please fix errors and try again.\nInvalid config for interface dp1\n",
  "prompt": "nsappliance"

Use radius server without the use of destination-networks. This means that radius-server can be configured as usual in the CLI. The “destination-networks” command should not be configured under interface configuration.

106506

ICAP client is currently not supported for the appliance.

There is no workaround.

116465

You cannot configure transaction event logs in this version of the appliance. Commands under fetch-web-proxy-logs are not supported.

set management-plane fetch-web-proxy-logs [...]

There is no workaround.
